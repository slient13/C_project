유도 자료형 : 배열, 포인터 등 기존 자료형으로 유도하는 자료형.

단축평가
    && 연산에 대해서 좌항의 값이 false 라면 우항의 연산은 실행하지 않음.
    || 연산에 대하여 좌항의 값이 true면 우항은 실행하지 않음.

형변환
    올림변환
        연산 등에서 데이터의 자료형이 상이한 경우, 보다 상위레벨의 자료형으로 변환되어 연산을 실행
    내림변환
        대상 자료형이 더 작은 범위인 경우, 일부 값을 버리고 대입시킴.

배열
    순서가 유의미한 자료형
    배열의 각 원소는 인접한 메모리에 위치함

포인터 배열
    int *data[3] : <int *> 타입 변수 3개로 구성된 배열

문자열 관련 내장함수 비교 : 
    getchar/putchar : 
        개요 : 입력을 받아 문자 1개를 반환.
        특징 : 
            라인 버퍼링 방식 이용.
            엔터를 입력하기 전까지 입력된 모든 값을 버퍼에 저장.
            버퍼에 저장된 값은 남아서 다음 'getchar()' 호출 시 자동으로 전달시킴.
    getche : 
        개요 : 입력을 받아 문자 1개를 반환.
        포함 헤더 : conio.h
        특징 : 버퍼를 사용하지 않고 입력을 즉각전달.
    getch/putch : 
        개요 : 입력을 받아 문자 1개를 반환.
        특징 : 
            키보드를 입력하는 순간 바로 전달.
            입력된 값을 보여주지 않음.
    gets/puts : 
        개요 : 한 줄의 문자열을 입력받아 전달.
        특징 : 
            엔터키 입력 전까지 모든 값을 버퍼에 저장.
            엔터키 입력 시, 버퍼에 있던 모든 값을 한번에 반환.
            문자열 맨 끝에 널 문자('\n')를 추가시켜줌.
            띄어쓰기가 있더라도 무시함.

struct(구조체) : 
    개요 : 커스텀 데이터타입을 생성
    특징 : 
        여러가지 데이터를 한데 묶어서 구현.
        맴버 변수들끼리 자료형이 같을 필요는 없음.
        맴버 변수로 다른 구조체를 사용 가능.
    문법 : 
        정의 : < struct <struct_name> {<variable>}; >
        선언 : < struct <struct_name> <variable> = {<value>, ...}; >
    클래스와의 차이점 : 
        기본 자료 공개형이 'public'.
        메소드를 가질 수 없음.
    연산자 : 
        = : 동일한 구조체의 변수들이면 가능. 각 요소들이 1:1 대응 됨.
        == : 비교 연산 사용 불가. 비교연산을 수행하려면 각 요소를 일일히 비교해야 됨.
    배열 : 
        가능 여부 : 가능.
    포인터 : 
        가능 여부 : 가능.
        비고 : 매개변수로 전달 될 때 값을 변경하고 싶다면 포인터나 참조형을 사용해야 됨.   
    포인터 맴버변수 : 
        가능 여부 : 가능.
